# خطة التنفيذ

## 1) زر "اقتراح فلتر بالذكاء الاصطناعي" في صفحة المتقدمين

- زر جديد في `ApplicantsAdvancedFilters.tsx` يفتح حوار: المستخدم يكتب وصفاً (مثل "محاسبين سعوديين خبرة 5 سنوات في الرياض").
- Edge function `ai-build-filters`: تستقبل الوصف + قائمة الحقول المتاحة + القيم الفعلية الموجودة في قاعدة البيانات (distinct values لكل عمود)، وتُرجع JSON منظم بقائمة فلاتر مقترحة.
- يعرض الحوار الفلاتر كبطاقات قابلة للتعديل/الحذف قبل الضغط "تطبيق على الفلتر"، ثم تنعكس على واجهة الفلتر الحالية.

## 2) مدير توحيد القيم (Synonyms + Normalize)

تبويب جديد في الإعدادات "توحيد القيم":
- يعرض كل القيم الفريدة لكل عمود (الجنسية، المدينة، التعليم، الوظيفة...) من جدول applicants مع عدد التكرار.
- زر "اقتراح بالذكاء الاصطناعي" يُجمّع المتشابهات تلقائياً (سعودي/Saudi/السعودية → سعودي).
- يمكن للمستخدم تعديل المجموعات والصيغة الموحدة قبل الحفظ.
- يُحفظ في جدول `value_synonyms` (موجود).
- **زر "تطبيق على البيانات الفعلية"** (اختياري):
  - يفتح حوار يعرض **ملخصاً تفصيلياً**: عدد السجلات المتأثرة لكل تحويل + عينة 5 صفوف.
  - يتطلب **PIN الحذف** الحالي + تأكيد كتابي "أؤكد" + توقيع نصي (اسم المستخدم).
  - عند التأكيد: edge function `apply-value-normalization` يُحدّث الأعمدة فعلياً، ويسجل العملية في `audit_log` مع نسخة احتياطية لكل سجل في `deleted_items` للاسترجاع.
  - بدون التطبيق على البيانات، طبقة المرادفات تكفي للعرض/التحليل (تعمل أصلاً).

## 3) ترتيب وتصنيف المسميات الوظيفية

تبويب جديد في الإعدادات "تنظيم الوظائف":
- يعرض كل المسميات الفريدة من `applicants.desired_position` + `recruitment_job_titles` مرتبة أبجدياً مع عدد التكرار.
- إضافة عمود `category` لجدول `recruitment_job_titles` + جدول جديد بسيط `job_categories` (هندسية، فنية، إشرافية، نظافة... قابل للتعديل).
- زر "تصنيف تلقائي بالذكاء الاصطناعي": edge function `ai-categorize-jobs` يصنف كل المسميات ويُرجع اقتراحاً.
- جدول مراجعة: كل مسمى مع الفئة المقترحة، يمكن تغييرها يدوياً، ثم "حفظ".
- عرض مجمّع حسب الفئة لتسهيل إيجاد المكررات ودمجها.

## التقنيات
- 3 edge functions جديدة باستخدام `google/gemini-2.5-flash` (Lovable AI).
- migration واحدة: عمود `category` + جدول `job_categories`.
- 3 مكونات React جديدة في `src/components/Dashboard/`.
- لن أمس البيانات الفعلية إلا عبر زر التطبيق المحمي بـ PIN + تأكيد + توقيع.
